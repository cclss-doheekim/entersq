# [PRD] 별(Byeol, 가칭) Phase 1: 베타 MVP

- 문서 버전: v0.3
- 작성 일시: 2026.04.15
- 이전 버전: v0.2 (2026.04.15), v0.1 (2026.04.06, "Planet")
- 상위 프로젝트: AllKpop
- **프로젝트명**: 별(Byeol) — **가칭**. 오픈 전 확정 필요

> **v0.2 → v0.3 주요 변경**
> - **IP 페이지 규모 확대**: "사전 신청 2~5개" → **주요 팬덤 IP 페이지 9~12개 선런칭** (BLINK, ARMY, MY, EXO-L, DIVE, MOA, Once, CARAT, NCTzen 등)
> - **프로필 온보딩 플로우** 구체화: 게임 닉네임 + **대표 팬덤(필수)** + IP 팔로우(선택, 복수)
> - **홈 메인뷰 = 별 세계 관전뷰**로 재정의 (게임 아님). 3D 공전 씬 + IP 랭킹 오버레이
> - **IP 페이지 = 3D 별 키우기 공간**: 씨앗/물/비료 액션 + 기여자 랭킹 오버레이
> - **미니게임 종료 UX**: 획득 포인트 → 별 선택 기여 → 랭킹 포맷 "[닉네임] [대표팬덤] N포인트를 [◯◯별]에 기여"
> - **뱃지/메달/온보딩 가이드 체계는 v0.1·v0.2 유지**

## 1. 제품 정의 및 목적

- **제품명**: 별 (Byeol) — **가칭**. AllKpop 팬덤 기여형 메타 게임 생태계
- **포지셔닝**: 베타 오픈. 정식 런칭 전 검증 단계
- **목적**: 올케이팝 유저에게 "성장·수집·팬덤 기여·경쟁"의 재미를 제공하여 체류시간을 늘리고, 미니게임 → 공통 성장 → 팬덤 기여 → 랭킹 경쟁으로 이어지는 순환 루프(Play–Earn–Contribute–Compete)를 검증
- **MVP 핵심 검증 포인트**:
  - 미니게임 플레이 → 공통 성장 지표 연결 시 반복 방문 발생 여부
  - **대표 팬덤 지정 + IP 별 기여 + 팬덤 간 경쟁 랭킹**이 체류시간을 유의미하게 끌어올리는가
  - 가벼운 성취 구조(뱃지/메달)가 초기 리텐션에 기여하는가
- **런칭 목표일**: 2026년 4월 22일 (베타)

## 2. 핵심 유저 스토리

1. **Play**: "나는 올케이팝의 가벼운 미니게임을 플레이하고 기록을 남기고 싶다."
2. **Earn**: "결과에 따라 경험치와 재화를 얻고 싶다."
3. **Contribute**: "내 결과를 내가 좋아하는 아티스트의 별에 기여하고 싶다."
4. **Compete** *(v0.3 신설)*: "내 기여가 팬덤 랭킹에 반영되어, 내 팬덤이 다른 팬덤보다 위에 오르는 걸 보고 싶다."
5. **Identity**: "나는 게임 전용 프로필에서 대표 팬덤과 기여도를 통해 내 정체성을 드러내고 싶다."
6. **Future Value**: "지금 모으는 재화/수집품/기여도가 이후 더 다양한 기능으로 확장될 것이라는 기대를 갖고 싶다."

## 3. 기능적 요구사항

### 3.1. 온보딩 및 프로필 설정 화면 *(v0.3 구체화)*

진입 조건: 올케이팝 계정 로그인 상태 + 별 전용 프로필 미존재 시 자동 진입.

- **Step 1 — 게임 닉네임**: 올케이팝 계정과 별도. 별 내부 식별자. 비밀번호 불요
- **Step 2 — 대표 팬덤 선택 (필수)**
  - 리스트 예: BLINK (BLACKPINK), ARMY (BTS), MY (Aespa), EXO-L (EXO), DIVE (IVE), MOA (TXT), Once (TWICE), CARAT (SEVENTEEN), NCTzen (NCT) 등
  - 1개만 선택. 프로필/랭킹 로그에 기본 노출
- **Step 3 — IP 팔로우 (선택, 복수 가능)**
  - 대표 팬덤과 별개로 관심 있는 IP 페이지 다중 팔로우
  - 홈에서 팔로우한 IP의 업데이트·랭킹을 우선 노출

프로필 페이지:
- 닉네임, **대표 팬덤 로고/배지**, 대표 뱃지, 획득 메달
- **당근온도형 IP 기여도** (예: "블랙핑크 55% / 아미 25% / 공용 20%")

### 3.2. 별 홈 화면 *(v0.3 재설계)*

#### 상단 메인뷰 — 별 세계 관전뷰
- v0.1/v0.2의 "메인 게임" 역할 제거 → **관전/로비 공간**으로 재정의
- 3D 씬: 올케이팝 주 행성을 중심으로 IP 별들이 공전
- IP 별의 크기·광도·장식은 해당 IP 포인트/레벨에 비례
- **오버레이 UI: IP 랭킹 Top N**
  - 항목 클릭 시 해당 **IP 페이지**로 이동
  - 실시간/근실시간 갱신

#### IP 페이지 리스트
- 존재하는 모든 IP 페이지 노출 (리스트 또는 카드형 게임 미리보기)
- 팔로우 중인 IP가 상단 우선

#### 미니게임 섹션
- 홈 하위에 배치. 썸네일 + 진입 CTA
- MVP 2~3종

### 3.3. IP 페이지 *(v0.3 신설/구체화)*

- **3D 별 메인뷰**: IP별 고유 3D 별 오브젝트. 테마 컬러/로고/배경 차별화
- **유저 액션**
  - 씨앗 심기 (Sparkle 소량 소비)
  - 물 주기 / 비료 주기 (소비 아이템 또는 데일리 쿨다운)
  - 각 액션은 별 포인트 가산 + 시각 변화(꽃/나무 성장)
- **기여 경로**
  - 미니게임에서 기여 지정 → 별 포인트 증가
  - 별 키우기 액션(심기/물/비료) → 별 포인트 증가
  - 양쪽 합산이 홈 IP 랭킹에 반영
- **오버레이 UI: 기여자 Top 랭킹**
  - 이 IP에 가장 많이 기여한 유저 Top N
  - 닉네임 + 누적 기여 포인트

### 3.4. 미니게임 *(v0.3 구체화)*

#### 선정 기준 (v0.2 유지)
- 올케이팝 IP/미디어/콘텐츠 연동 필수
- 짧은 학습곡선, 무한 반복, 명확한 점수 산출

#### 후보 예시
- 올케이팝 기사 속 보석 찾기 (기사 체류 유도)
- 케이팝 음원 맞추기 (N초 듣기) — 저작권 검토 필요
- 케이팝 가사 단어 맞추기
- 가벼운 아케이드 1종 병행 고려

#### 게임 종료 플로우 *(v0.3 구체화)*
1. **사용자 랭킹 표시** (이번 게임 스코어 기반, 개인 랭킹)
2. **기여 선택 다이얼로그**
   - "획득한 N포인트를 어디에 기여할까?"
   - 옵션: 팔로우 중인 IP 별 + 대표 팬덤 별 + 공용 올케이팝 별
   - 기본값: 대표 팬덤 별
3. **기여 확정** → 선택된 별 포인트 상승 + 홈 IP 랭킹 반영
4. **랭킹 피드 로그 포맷** *(v0.3 신설)*:
   ```
   [닉네임] [대표팬덤] N포인트를 [◯◯별]에 기여
   ```
   - 전역 랭킹 피드와 해당 IP 페이지의 기여 히스토리에 동시 노출
   - 1등 로그는 하이라이트 처리하여 경쟁심리 자극

### 3.5. 보상 및 성장 엔진

- **Sparkle (공통 재화)**: 게임 결과 비례 지급. 미션 보너스. 심기/물주기 등에 소량 소비
- **Planet EXP**: 개인 경험치 + 별 경험치 (양쪽에 비례 가산)
- **Artifact (수집품)**: 씨앗·꽃·나무 등 세계관 아이템. 별에 심기 가능
- **인벤토리**: Artifact + 소비 아이템(입장권 등) 리스트 뷰

### 3.6. 세계관

- IP 부재 대응. 별 세계 — 행성 공전, 씨앗 → 물·비료 → 꽃·나무 성장
- 시즌별 매개체 교체 가능성 유지 (별 → 섬 → 바다)
- 최종 성장형 비주얼은 "키우는 맛" 위해 화려하게

### 3.7. 가이드·메달·뱃지 (v0.2 유지) *(v0.3 유지 명시)*

- **온보딩 가이드 (예시)**
  - 게임 닉네임 설정
  - 대표 팬덤 선택
  - 별 EXP 쌓기
  - 미니게임 1회 플레이
  - 별 공유하기
  - IP 1개 이상 팔로우
- **메달**: 가이드/활동 완료 시 즉시 해금, 프로필에서 확인
- **메달 설계 원칙**: 이름 있는 개별 메달, 즉각적 성취 피드백, 서비스 사용법 자연 학습
- **뱃지**: 신규회원 → 루키 → 파이어니어 → 익스플로러 → 커맨더

### 3.8. 미션 및 사회성

- **리더보드**
  - 개인 랭킹 (주간 단위 게임 스코어)
  - IP 랭킹 (홈 메인뷰 오버레이)
  - IP별 기여자 Top 랭킹 (IP 페이지 오버레이)
- **공유 기능**: 내 별 / 기여 성과 외부 SNS 공유 (공유 시 보상)

## 4. 운영 및 관리 요구사항 (Admin)

- **IP 페이지 메타 관리**: 9~12개 IP 페이지의 테마 이미지·로고·배경·별 3D 에셋 슬롯 관리 (MVP는 하드코딩 수준 수동 관리 허용)
- **배너/팝업 광고 구좌** (MVP 1차 광고 상품)
- **지표 로그**: 게임별 플레이 횟수, 유저별 재화 획득/소비, 팬덤 기여 로그, 랭킹 변동 이벤트 적재
- **랭킹 피드 관리**: 부적절 닉네임 필터, 노출 가이드라인

## 5. 핵심 성과 지표 (KPI)

- **평균 세션 길이** (목표: 포럼 대비 +20%)
- **재방문율 D1/D7**
- **교차 플레이율**
- **Sparkle Burn Rate** *(v0.1 오타 Stardust → Sparkle)*
- **팬덤 지표**
  - 대표 팬덤 지정 완료율 (온보딩 완료율)
  - 평균 IP 팔로우 수
  - 팬덤 기여 지속성
  - 팬덤 밸런스
- **경쟁·랭킹 지표** *(v0.3 신설)*
  - 랭킹 노출 후 세션 연장률
  - 랭킹 항목 클릭→IP 페이지 이동률
  - 1등 변동 빈도와 변동 직후 기여 액션 증가율

## 6. 미해결 논의 사항

- **프로젝트 정식 명칭** (가칭 "별" 확정 여부)
- **GA 전략**: 통합 / 전용 / 합산
- **저작권 범위**: 음원·가사·기사 이미지
- **IP 페이지 최종 라인업**: 편집 추천 / 사전 신청 집계 / 혼합
- **3D 에셋 제작 방식**: 외주 / AI + 후가공 / 내부 (디자이너 부재)
- **대표 팬덤 변경 정책**: 주기 제한, 쿨다운, 변경 시 누적 기여도 처리
- **공용 "올케이팝 행성" 역할**: 관전 중심지 전용 / 중립 기여처 겸용

## 7. 제약 사항 및 우선순위

- **High**
  - 온보딩 플로우(닉네임 + 대표 팬덤 + IP 팔로우)
  - 별 홈 관전뷰 + IP 랭킹 오버레이
  - IP 페이지 (3D 별 + 씨앗/물/비료 + 기여자 랭킹)
  - 미니게임 종료 후 기여 선택 + 랭킹 피드 포맷
  - 게임 프로필, 인벤토리, 온보딩 가이드·메달·뱃지
- **Medium**
  - 일일/주간 미션, 보상 지급 로직
  - 미니게임 2~3종 (IP/미디어 연동)
  - 당근온도형 기여도 프로필 표시
  - 배너/팝업 광고 구좌
  - 외부 SNS 공유
- **Low**
  - 소셜 인터랙션(친구 방문 등) — Phase 4
  - 팬덤 경쟁 프리미엄 광고 구좌 — Phase 4
  - 보상형 리워드 광고 — Phase 2
