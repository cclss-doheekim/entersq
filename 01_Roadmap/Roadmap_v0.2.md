> - **시즌 매개체 교체** 개념 추가 (시즌1 행성 → 시즌2 섬 → 시즌3 바다 식으로, 운영 비용 절감 + 신선도 유지)
> - **미디어/콘텐츠 연동**을 미니게임 선정 필수 조건으로 격상 (올케이팝 기사 보물찾기, 음원 맞추기 등)

## 1. 프로젝트 배경 및 전략적 방향

별(Byeol)은 올케이팝의 "읽기" 중심 유저 경험을 "참여·체류" 중심으로 전환하기 위한 서브 프로젝트입니다. 텍스트 포럼 형식에서 벗어나, 유저가 **자신이 좋아하는 아티스트의 별을 함께 키우는** 팬덤 기여형 메타 게임 생태계를 목표로 합니다.

- **팬덤 중심 루프**: 개인의 게임 활동이 자동으로 자신이 지지하는 아티스트 별의 성장에 기여되는 구조. 경쟁과 응원이 자연스럽게 발생
- **통합 경제 + 분기된 성취**: Sparkle/EXP/Artifact는 공통 화폐/지표로 운영하되, 성취는 "올케이팝 공용 별"과 "아티스트 별" 두 층위로 분기
- **간접 기여 KPI**: 직접 매출보다는 체류시간·재방문·세션당 게임 수를 핵심 지표로 설정. 베타 단계에서 제품 생존력을 먼저 검증
- **IP 공백 대응 세계관**: 우리는 IP를 소유하지 않으므로, 자체 세계관(씨앗/물/꽃/나무 등의 성장 메타포)으로 몰입을 보완

## 2. 단계별 구축 로드맵

### Phase 1. 베타 MVP 출시: 팬덤 기여 루프 검증 (4월 22일)

**MVP 목표**: 게임피케이션이 올케이팝 체류시간을 유의미하게 올리는가 + 팬덤 기여 루프가 재방문을 유도하는가 검증.
정식 오픈이 아닌 **베타 오픈**이며, 지표 반응에 따라 이후 단계의 속도·범위를 조절합니다.

- **공용 별 허브(Byeol Hub) 구축**
  - 올케이팝 공용 별이 메인 허브 역할. 전체 유저 활동의 시각적 집결지
  - 상단 대시보드에 보유 Sparkle, Planet EXP, 레벨 상시 노출
  - 미니게임 진입점 + 프로필/랭킹/가이드 이동 링크
- **아티스트 별 (사전 수요 기반 2~5개)** *(v0.1 대비 신설)*
  - 오픈 전 "어떤 아티스트 별을 열어드릴까요?" 사전 신청 이벤트 운영
  - 응답 상위 2~5개 아티스트 별을 MVP에 선런칭 (예: 블랙핑크별, BTS별 등)
  - 공용 별과 동일한 UI 템플릿 적용, 차이는 테마/이름/리더보드 분리
- **팬덤 팔로우 & 기여 지정** *(v0.1 대비 신설)*
  - 유저는 여러 아티스트 별을 팔로우 가능 (복수 허용)
  - 게임 종료 시 "이 점수를 어디에 기여할래?" 선택 → 선택된 별에 점수 가산
  - 당근 온도형 개인 기여도 프로필 노출 ("블랙핑크 기여 60% / 아미 기여 30%")
- **공통 보상 및 성장 체계 확립**
  - Sparkle(재화): 게임 플레이로 획득. MVP는 적립 중심, 소비처 최소
  - Planet EXP(경험치): 별의 레벨을 높임
  - Artifact(수집품): 씨앗·꽃·나무 등 세계관 아이템. 유저가 특정 별에 "심기" 가능
- **게임 전용 프로필 및 성취 구조**
  - 게임용 닉네임만 설정 (비밀번호 불필요, 올케이팝 계정 연동) *(v0.1에서 변경)*
  - 뱃지: "신규회원" 중심 + 활동에 따라 탐험가→파이어니어→익스플로러→커맨더 등급 변화
  - 메달: 가이드 미션 완료 시 즉각 해금
- **가이드 기반 초기 온보딩**
  - 예: 별 EXP 쌓기 / 별 공유하기 / 미니게임 1회 / 팬덤 팔로우 1개 선택 *(마지막 항목 신설)*
- **미니게임 콘텐츠 (2~3종)**
  - **필수 기준: 올케이팝 IP/미디어/콘텐츠와 엮임**
  - 후보: 올케이팝 기사 속 "보석 찾기"(기사 내 랜덤 다이아몬드 클릭 → Sparkle 획득으로 기사 체류 증가), 케이팝 음원 맞추기, 케이팝 가사 단어 맞추기 등 (저작권 검토 필요)
  - 학습곡선 짧고 무한 반복 가능, 명확한 점수 산출
- **세계관 설정** *(v0.1 대비 신설)*
  - IP가 없는 공백을 메꿀 자체 세계관. 성장 메타포는 씨앗→물/비료→꽃/나무
  - 시즌 매개체 교체 가능성 명시 (이번 시즌은 "별", 다음 시즌은 "섬"으로 전환 가능)
- **운영 및 광고 구좌**
  - 이벤트 배너 + 팝업 구좌 MVP 포함 (배너·팝업은 기존 사업 모델에서 가장 비싸게 판매 가능한 구좌)
  - "팬덤 1등 아티스트 노출" 광고 구좌는 Phase 2~3 이후로 후순위

### Phase 2. 리텐션 강화: 시즌제 + 소비처 설계 (5월)

- **시즌제 구조 본격화**
  - 시즌 길이는 월간 또는 분기 단위로 탄력 운영. 시즌별 한정 보상과 미션
  - 시즌 종료 시 "별 다이어리" 자동 생성 (시즌 활동 스크린 캡처 모음 — v0.1에서 계승)
  - **시즌 매개체 교체 실험**: 시즌1 별 → 시즌2 섬 → 시즌3 바다 식으로 매개체를 바꿔 신선도 유지
- **순환형 소비처 설계**
  - Sparkle을 꽃·나무 심기, 별 꾸미기, 레벨업 가속, 한정 스킨 등에 사용
  - "획득 → 축적 → 사용 → 자랑"의 기본 루프 완성
  - 심기 행위에 팬덤 기여가 자동 연동 (예: "김도이 님이 블랙핑크 별에 장미를 심었습니다")
- **업적·사회적 증명**
  - 개인 랭킹 + 팬덤별 기여도 랭킹 병행
  - 누적 플레이/기여 기반 업적 시스템
- **보상형 광고 실험**
  - 유저 경험 해치지 않는 선에서 입장권/부스트 지급형 리워드 광고
- **시각적 고도화**
  - 별 레벨에 따라 외형 진화, 장식 슬롯 확장

### Phase 3. 플랫폼화 준비: 확장성 및 아티스트 별 대량 운영 (6월 초)

- **미니게임 삽입 규격(SDK/API) 정의**
  - 신규 게임 추가 시 보상 지급, 결과 리포트, 광고 호출 자동 연동
- **아티스트 별 템플릿 + 백오피스**
  - 신규 아티스트 별을 운영팀이 자체 생성·설정 가능하도록 관리도구 고도화
  - 썸네일, 설명, 테마 아이템, 노출 우선순위 제어
- **운영 정책 모듈화**
  - 게임별 보상 밸런스, 플레이 제한값, 시즌 매개체 전환 등을 실시간 반영
- **외부 협업 모델 검증**
  - 외부 파트너/외주 게임 1종 이상 연동해 플랫폼 확장성 기술 검증
- **아티스트 전용 이벤트 상품화**
  - 특정 아티스트를 위한 이벤트성 미니게임 개별 제작 = 사업부 신상품
  - 이 경우 별도 비용 청구 모델 수립

### Phase 4. 팬덤 심화 및 광고 고도화 (6월 말)

- **공동 목표/협동 이벤트**
  - 팬덤별 "월드 퀘스트" — 일정 기간 집합 목표 달성 시 팬덤 전원 보상
- **아티스트 테마 심화**
  - 아티스트별 전용 수집품, 시그니처 아이템, 컴백 연동 이벤트
- **소셜 인터랙션**
  - 서로의 별 방문·응원·리액션, 외부 SNS 공유 최적화
- **팬덤 경쟁형 광고 구좌 출시** *(미팅 논의 반영)*
  - "이 주의 1등 팬덤 메인 배너" — 팬덤에게 직접 판매 가능한 프리미엄 구좌
  - 지하철 광고판 메타포로 브랜딩
- **네이티브 광고 (Sponsored Objects)**
  - 별 내부 환경과 어우러지는 브랜드/아티스트 오브젝트

## 3. 핵심 성과 지표 (KPI)

베타 단계 핵심은 "게임피케이션이 올케이팝 체류를 유의미하게 올리는가"의 검증입니다.

- **체류 지표 (최우선)**: 평균 세션 길이, 1인당 평균 플레이 게임 수, 올케이팝 포럼 대비 체류 증분
- **충성도 지표**: D1/D7 재방문율, 미션·업적 달성률, 팬덤 팔로우 유지율
- **팬덤 기여 지표** *(v0.1 대비 신설)*: 팬덤 지정율(프로필 중 팬덤 팔로우 1개 이상 보유 비율), 팬덤별 기여 균형도, 팬덤 변경 빈도
- **경제 지표**: 재화 획득 후 사용률(Burn Rate), 광고 참여(Opt-in) 비중
- **플랫폼 지표**: 신규 게임 추가 리드타임, 교차 플레이율, 신규 아티스트 별 추가 리드타임

## 4. 네이밍 (가칭 운영)

- 현재 작업명: **별(Byeol)** — 가칭
- 논의된 후보: 별, 아일랜드, 섬, Orbit, Bloom, Lumo, 누리, 소리, Riff, Popping, Playground/Arcade/Lounge 계열
- 보류/제외 근거: "Planet"은 팬덤 명칭과 중첩될 여지(플래닛), "Verse/Orbit"은 위버스·우주 단어 경쟁 우려, "Lumo"는 킥보드 브랜드 연상
- 매개체와 이름의 분리 가능성: 시즌별로 매개체(별→섬→바다) 교체 운영이 가능하므로, 서비스명은 매개체와 독립적인 브랜드 네임으로 두는 것도 옵션
- 확정 시점: MVP 베타 오픈 전 네이밍 확정 + 로고/워드마크 최소본 필요

## 5. 미해결 논의 사항 (Open Questions)

v0.1에서는 명시되지 않았던 미팅 중 과제. Phase 1 오픈 전 결정 필요.

- **GA 전략**: 올케이팝 GA에 통합 / 별도 GA로 분리 / 합산 리포트 구성 중 택1. 유입 전략과 퍼널 분석 방향에 따라 결정
- **저작권 범위**: 미니게임에 음원/가사/기사 이미지 사용 가능 범위 법무 검토
- **직접 매출 기여 여부**: 베타에서는 간접 기여(체류→광고매출)에 포커스. 직접 매출(결제) 도입 시점 판단 필요
- **사전 신청 아티스트 별 선정 기준**: 신청 수 상위 / 편집 추천 / 혼합 방식 중 택1
- **디자인 리소스**: 디자이너 부재 상태. 외주 / 내부 겸업 / AI 에셋 활용 중 선택 필요 (최종 별 외형은 "키우는 맛"을 위해 화려해야 함)
