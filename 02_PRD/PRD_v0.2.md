# [PRD] 별(Byeol, 가칭) Phase 1: 베타 MVP

- 문서 버전: v0.2
- 작성 일시: 2026.04.15
- 이전 버전: v0.1 (2026.04.06, 프로젝트명 "Planet")
- 상위 프로젝트: AllKpop
- **프로젝트명**: 별(Byeol) — **가칭**. 정식 명칭 미정, 오픈 전 확정 필요

> **v0.1 → v0.2 주요 변경**
> - 프로젝트명 **Planet → 별(Byeol, 가칭)**
> - 포지셔닝을 **"베타 오픈"** 으로 명시 (그랜드 런칭 X, 검증 목적)
> - **아티스트 별 구조**를 MVP에 포함 (v0.1에서는 Phase 4 항목이었음)
> - **팬덤 팔로우 & 기여 지정** 플로우 신규 도입
> - **게임 전용 프로필의 비밀번호 항목 제거** (닉네임만)
> - **세계관(씨앗/물/꽃/나무)** 설정 신규 추가
> - 미니게임 선정 기준에 **"IP/미디어/콘텐츠 연동 필수"** 를 명문화
> - KPI에 **팬덤 기여 지표 계열** 추가
> - **이벤트 배너/팝업 광고 구좌** MVP 범위로 확정 명시
> - KPI 항목의 오타(Stardust) 수정 → Sparkle로 통일

## 1. 제품 정의 및 목적

- **제품명**: 별 (Byeol) — **가칭**. AllKpop 팬덤 기여형 메타 게임 생태계. 정식명은 오픈 전 확정 예정
- **포지셔닝**: 베타 오픈. 정식 런칭 전 검증 단계
- **목적**: 올케이팝 유저에게 "성장·수집·팬덤 기여"의 재미를 제공하여 체류시간을 늘리고, 미니게임→공통 성장→팬덤 기여로 이어지는 최소 순환 루프(Play–Earn–Contribute)를 검증
- **MVP 핵심 검증 포인트**:
  - 미니게임 플레이가 공통 성장 지표와 연결될 때 반복 방문이 발생하는가
  - **내가 좋아하는 아티스트의 별에 기여한다**는 동기가 체류를 끌어올리는가 *(신설)*
  - 프로필/뱃지/메달의 가벼운 성취가 초기 리텐션에 기여하는가
- **런칭 목표일**: 2026년 4월 22일 (베타)

## 2. 핵심 유저 스토리

1. **Play**: "나는 올케이팝의 가벼운 미니게임을 플레이하고 기록을 남기고 싶다."
2. **Earn**: "결과에 따라 내 별을 키울 수 있는 경험치와 재화를 얻고 싶다."
3. **Contribute** *(신설)*: "내 게임 결과를 내가 좋아하는 아티스트의 별에 기여하고, 그 팬덤의 성장에 일조하고 싶다."
4. **Identity/Progression**: "나는 게임 전용 프로필을 만들고, 뱃지·메달·기여도를 통해 내 정체성과 성취를 보여주고 싶다."
5. **Future Value**: "지금 모으는 재화·수집품·기여도가 이후 더 다양한 기능으로 확장될 것이라는 기대를 갖고 싶다."

## 3. 기능적 요구사항 (Functional Requirements)

### 3.1. 메인 시스템: 별 허브 (Byeol Hub)

- **공용 별(올케이팝 별) 메인 뷰**: 전체 유저의 활동이 시각적으로 집결. 레벨, 장착 Artifact, 심어진 수집품을 표시
- **아티스트 별 탭/리스트** *(신설)*: 공용 별과 동일 UI 템플릿의 2~5개 아티스트 별 (사전 신청 상위). 탭 또는 리스트형 네비게이션으로 전환
- **대시보드**: 보유 Sparkle, Planet EXP 게이지, 현재 레벨을 상단에 상시 노출
- **진입점**: 제공되는 2~3종 미니게임으로 바로 진입. 프로필/랭킹/가이드 이동 가능

### 3.2. 계정 및 프로필 시스템

- **게임 전용 프로필 생성**:
  - 첫 진입 시 게임용 **닉네임만** 설정 *(v0.1: 비밀번호 검토 → v0.2: 불필요로 확정)*
  - 올케이팝 로그인 계정과 자동 연동, 게임 내 식별자로 사용
- **팬덤 팔로우** *(신설)*:
  - 프로필에서 팬덤(= 아티스트 별) 복수 팔로우 가능
  - 팔로우한 팬덤 중 하나를 "대표 팬덤"으로 지정 가능 (프로필에 로고 노출)
- **프로필 페이지**:
  - 닉네임, 대표 뱃지, 획득한 메달 리스트, 기본 활동 요약
  - **팬덤 기여도(당근 온도식 표기)** — 예: "블랙핑크 기여 60% / 아미 기여 30% / 공용 별 10%" *(신설)*
- **초기 뱃지**:
  - MVP는 기본 뱃지 중심 (예: 신규회원)
  - 활동에 따라 루키 → 파이어니어 → 익스플로러 → 커맨더 등급 전이

### 3.3. 보상 및 성장 엔진

- **공통 재화 (Sparkle)**:
  - 미니게임 결과(점수)에 비례 지급
  - 일일/주간 미션 완료 시 보너스
  - MVP는 직접 소비처 최소 (적립 중심). 단, 아래 "심기" 액션에 소량 소비 가능
- **성장 지표 (Planet EXP)**:
  - 게임 플레이 및 활동 포인트에 따라 누적
  - 레벨업 시 시각 이펙트 + 해금 요소
- **팬덤 기여 엔진** *(신설)*:
  - 게임 종료 시 "점수를 어디에 기여할까?" 선택 다이얼로그 노출
  - 선택된 별(공용 별 또는 팔로우한 아티스트 별)에 점수 가산 → 해당 별의 EXP 상승
  - 개인 기여도 기록 누적 → 프로필 당근 온도에 반영
- **수집품 (Artifact) 시스템**:
  - 씨앗·꽃·나무 등 세계관 기반 아이템 (장식/배지 겸용)
  - **심기 액션**: 보유한 씨앗/꽃을 특정 별에 심을 수 있음 (기여도 + 업적 로그 동시 발생)
  - 예: "김도이 님이 블랙핑크 별에 장미를 심었습니다"
- **성취 요소**: 메달
- **인벤토리 시스템**: 보유 Artifact와 소비 아이템(입장권 등)의 기본 리스트 뷰

### 3.4. 세계관 설정 *(신설)*

- IP가 없는 공백을 자체 세계관으로 보완
- 성장 메타포: **씨앗 → 물·비료 → 꽃·나무 → 별의 장식 진화**
- 시즌 매개체 교체 가능성 유지 (시즌1 "별" / 추후 "섬" "바다" 등으로 전환 여지)
- 비주얼은 최종 성장형이 화려해야 함 ("키우는 맛" 필수)

### 3.5. 가이드 및 메달 시스템

- **가이드 예시**:
  - 별 EXP 쌓기
  - 별 공유하기
  - 미니게임 1개 플레이
  - **팬덤 팔로우 1개 선택** *(신설)*
- **메달 시스템**:
  - 가이드 또는 특정 활동 완료 시 즉시 해금
  - 프로필에서 확인 가능
- **메달 설계 원칙**:
  - 이름 있는 개별 메달 형태
  - 즉각적 성취 피드백
  - 초기 유저가 서비스 사용법을 자연스럽게 익히도록 유도

### 3.6. 미션 및 사회성 기능

- **리더보드 (Ranking)**:
  - 주간 단위 게임 점수 랭킹 (개인)
  - 전체 별 레벨 기반 누적 랭킹 (별 단위)
  - **팬덤별 기여 랭킹** *(신설)*: 이번 주 블랙핑크 1위 / BTS 2위 등
- **공동 기여도 표시**:
  - 유저들이 획득한 점수의 합계를 별 단위로 집계 시각화
  - "우리가 함께 이 별을 키우고 있다"는 감각 전달
- **공유 기능**: 내 별 외부 SNS 공유 (공유 시 씨앗 등 보상)

### 3.7. 미니게임 콘텐츠 (MVP 2~3종)

- **선정 기준** (우선순위 순):
  1. **올케이팝 IP/미디어/콘텐츠와의 연동 필수** *(미팅 반영 - 성민 의견)*
  2. 짧은 학습 곡선, 무한 반복 가능성
  3. 명확한 점수 산출
- **후보 예시** *(구체화됨)*:
  - **올케이팝 기사 속 보석 찾기**: 특정 기사 내 랜덤 다이아몬드를 찾으면 Sparkle 지급. 기사 체류/소비 동시 유도
  - **케이팝 음원 맞추기** (N초 듣고 곡명·아티스트 맞추기) — 저작권 검토 필요
  - **케이팝 가사 단어 맞추기** (빈칸 채우기형)
  - 체류 시간 주 KPI 감안, 테트리스급 가벼운 아케이드 게임도 1종 병행 고려
- **데이터 연동**: 게임 종료 시 결과값(Score)을 별 서버로 전송 → 보상 + 팬덤 기여 다이얼로그 진입

## 4. 운영 및 관리 요구사항 (Admin)

- **사전 신청 이벤트 운영** *(신설, 런칭 전)*: "어떤 아티스트의 별을 열어드릴까요?" 폼 + 랜덤 추첨 이벤트
- **배너 및 공지 관리**: 별 메인 내 이벤트 배너/팝업 게시·관리
- **광고 구좌** *(명시됨)*: 메인 배너 + 팝업 = MVP 1차 광고 상품
- **기본 지표 로그**: 게임별 플레이 횟수, 유저별 재화 획득/소비량, 팬덤 기여 로그 적재
- **아티스트 별 메타 관리**: 썸네일, 설명, 테마 이미지 수동 입력 (MVP는 2~5개 하드코딩 수준 허용)

## 5. 핵심 성과 지표 (KPI) 및 측정 계획

- **평균 세션 길이**: 별 진입 후 이탈 전까지 머무는 시간 (목표: 기존 포럼 대비 +20%)
- **재방문율 (D1/D7 Retention)**: 첫 플레이 후 다음 날/다음 주 재플레이 비율
- **교차 플레이율**: 제공된 2~3종 게임을 모두 플레이해 본 유저 비중
- **재화 소비율 (Burn Rate)**: 획득한 **Sparkle** 중 심기/가속에 소비된 비율 *(v0.1 오타 "Stardust" → Sparkle로 수정)*
- **팬덤 지표** *(신설)*:
  - **팬덤 지정율**: 팬덤 팔로우 1개 이상 보유한 유저 비율
  - **팬덤 기여 지속성**: 동일 팬덤에 반복 기여하는 유저 비율
  - **팬덤 밸런스**: 팬덤별 기여 분포 편중도

## 6. 미해결 논의 사항 (Open Questions)

- **프로젝트 정식 명칭 확정**: 현재 "별(Byeol)"은 가칭. 오픈 전 최종 네이밍 + 로고 확정 필요
- **GA 전략**: 올케이팝 GA 통합 / 별 전용 GA / 합산 — 결정 필요
- **저작권**: 음원·가사·기사 이미지 활용 범위 법무 검토
- **사전 신청 아티스트 선정 방식**: 신청 수 기준 / 편집 추천 / 혼합
- **디자인 리소스 확보 방안**: 외주 / 내부 겸업 / AI 에셋

## 7. 제약 사항 및 우선순위

- **High**: 공용 별 허브 뷰, 아티스트 별(2~5개), 팬덤 팔로우/기여 지정 플로우, 게임 프로필, 인벤토리, 가이드 *(아티스트 별·기여 플로우가 v0.1 대비 High로 승격됨)*
- **Medium**: 일일/주간 미션, 기본 보상 지급 로직, 미니게임 2~3종 연동, 당근온도형 기여도 프로필 표시, 배너/팝업 광고 구좌
- **Low**: 소셜 인터랙션(친구 방문 등 — Phase 4로 이관), 팬덤 경쟁 프리미엄 광고 구좌(Phase 4), 보상형 광고(Phase 2)
